# /kb コマンドリファレンス

`/kb`（kokonatu backup）コマンド群は、Discordサーバーの設定を丸ごとバックアップし、後から復元する機能を提供します。  
すべてのコマンドは **サーバーオーナーのみ** 実行できます。

---

## バックアップできる内容

| 項目 | 説明 |
|---|---|
| **ロール** | 名前・色・権限・位置を含むすべてのロール |
| **カテゴリ** | カテゴリの名前・順序・権限 |
| **チャンネル** | テキスト・ボイスチャンネルの名前・設定・権限 |
| **絵文字** | カスタム絵文字（画像データを含む） |
| **AutoMod** | Discordネイティブ AutoMod ルール |
| **オンボーディング** | サーバーオンボーディング設定 |

---

## コマンド一覧

### `/kb backup [notes]`

サーバーの現在の状態をバックアップし、**復元ID（UUID）**を発行します。

| 引数 | 説明 |
|---|---|
| `notes`（任意） | バックアップのメモ（例：`設定変更前`、`ロール整理前`）|

- 実行後に復元IDが表示されます。このIDは `/kb restore` で使用します。
- 絵文字の画像データも自動でダウンロード・保存されます。
- 同時に別の処理が実行中の場合はエラーになります。

```
/kb backup notes:ロール整理前
```

---

### `/kb check`

このサーバーの**最新バックアップの状態**を確認します。

引数は不要です。以下の情報が表示されます。

- 最新の復元ID（一部マスク表示）
- バックアップの作成日時
- メモ
- カテゴリ・チャンネル・ロール・絵文字の件数

```
/kb check
```

---

### `/kb info <id>`

指定した復元IDの**詳細情報**を表示します。

| 引数 | 説明 |
|---|---|
| `id` | 復元ID（UUID）。`/kb list` で確認できます |

- バックアップの作成者・日時・バックアップ内容を詳しく表示します。
- 画面下部に `/kb restore id:<ID>` のヒントが表示されます。

```
/kb info id:xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

---

### `/kb list`

このサーバーの**バックアップ履歴**をすべて表示します。

引数は不要です。以下の情報が一覧で表示されます。

- 復元ID（完全なUUID）
- 作成日時
- メモ
- ファイルサイズ（MB）

```
/kb list
```

---

### `/kb restore <id>`

指定した復元IDからサーバーを復元します。

| 引数 | 説明 |
|---|---|
| `id` | 復元ID（UUID）。`/kb list` で確認できます |

#### 実行前の確認事項

復元を実行する前に、以下の条件を満たしている必要があります。

| 条件 | 詳細 |
|---|---|
| **クールダウン** | 前回の復元から **3日間** が経過していること（不正利用防止） |
| **Botのロール位置** | Botのロールが**サーバー内の最上位**であること |

Botのロールが最上位でない場合、上位にあるロールのメンバーに対してロール操作ができないためエラーになります。  
サーバー設定 → ロール から、Guardianのロールを最上位に移動してください。

#### 復元フロー

1. 実行するとバックアップ内容が表示されます
2. **除外したい項目**（ロール・チャンネル等）をUIで選択できます
3. 確定すると復元処理が開始されます

:::caution 注意
復元は既存の設定を上書きします。必要に応じて事前に `/kb backup` で現在の状態をバックアップしてください。
:::

```
/kb restore id:xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

---

### `/kb share`

このサーバーの**最新バックアップIDを共有**できる形で表示します。

引数は不要です。復元IDがコードブロック形式で表示されるので、コピーして第三者に渡せます。

- 最新の復元ID（完全なUUID）
- バックアップ総数
- 最新バックアップの作成日時・サーバー名

```
/kb share
```

---

## 制限・注意事項

| 項目 | 制限 |
|---|---|
| **復元クールダウン** | 3日に1回（72時間） |
| **同時実行** | 1サーバーにつき1処理のみ（ロック制御） |
| **権限** | サーバーオーナーのみ実行可能 |
| **Botロール位置** | 復元時は最上位ロールが必要 |

---

## 推奨ワークフロー

```
1. 大きな設定変更の前に /kb backup notes:変更前 を実行
2. 変更後に問題が発生したら /kb list で復元IDを確認
3. /kb restore <id> で元の状態に戻す
```

設定変更の都度バックアップを取っておくことで、いつでも以前の状態に戻せます。
