# アクションモード

guardian の検知系機能（コンテンツフィルタ・スパム・招待フィルタ）は、違反時の処理をアクションモードで細かく制御できます。

---

## モード一覧

| モード | 削除 | チャンネル通知 | ポイント加算 | ログ記録 | セキュリティアラート |
|---|:---:|:---:|:---:|:---:|:---:|
| `delete_notify` | ✅ | ✅ | ✅ | ✅ | ✅ |
| `delete_silent` | ✅ | ❌ | ✅ | ✅ | ✅ |
| `warn_only` | ❌ | ✅ | ✅ | ✅ | ✅ |
| `log_only` | ❌ | ❌ | ❌ | ✅ | ❌ |

- **`delete_notify`**（デフォルト）: メッセージを削除し、チャンネルに警告を送信します。ポイントも加算されます。
- **`delete_silent`**: メッセージを黙って削除します。チャンネルへの通知はありませんが、ポイントは加算されます。
- **`warn_only`**: メッセージは削除しませんが、警告を送信しポイントを加算します。
- **`log_only`**: ログ記録のみ行います。削除・通知・処罰は一切しません。監視・テスト用途に適しています。

:::note チャンネル通知について
`delete_notify` / `warn_only` のチャンネル通知は、8〜10 秒後に自動削除されます。
:::

---

## アクションモード設定対象

| 設定フラグ | 対象検知 |
|---|---|
| `cf_action_content_limit` | 行数・文字数・メンション超過 |
| `cf_action_attachments` | 添付ファイル違反（数・拡張子・サイズ）|
| `cf_action_links` | リンクフィルタ違反 |
| `cf_action_keywords` | キーワードフィルタ違反 |
| `spam_action_solo` | 単独スパム（高速連投）違反 |
| `spam_action_cluster` | 協調スパム（クラスター）違反 |
| `invite_action` | 招待リンクフィルタ違反 |

---

## 変更コマンド

```
/guardian set-action <対象> <モード>
```

例: キーワード違反を `log_only`（監視のみ）に設定する

```
/guardian set-action キーワード違反 log_only
```

---

## カスタム通知プレフィックス

チャンネル通知メッセージの冒頭にカスタムテキストを付与できます。

```
/gs notify-prefix [prefix]
```

例: `[自動] ` を設定すると、通知が `[自動] @username 禁止ワードが...` のようになります。  
引数なしで実行すると、プレフィックスを解除します。
